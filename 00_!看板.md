## 命名规则

| 号段 | 用途 | 示例 |
|---|---|---|
| `00_!看板.md` | 全局看板（文件） | 本页 |
| `00_想法&诉求池/` | 演化起点 | — |
| `20_产品/` | 产品 wrapper | 内部 `CP{NN}_{名称}_{状态}`，如 `CP01_xxx_已上线` |
| `10_项目/` | 项目 wrapper | 内部 `XM{NN}_{名称}_{状态}`，如 `XM01_xxx_进行中` |
| `30_迭代管理/` | 运行期高频：发版打包视图 | `2026-W18.md` |
| `40_线上问题/` | 运行期事件：线上 issue 队列 | 月归档 `2026-04/...` |
| `8X_*` | 横切参考类 | `80_研究沉淀/` `81_知识库/` |
| `9X_*` | 系统配置 | `99_系统/`（模板/规则/提示词） |

> CP/XM 编号一旦分配永不复用，即使消亡也保留编号，避免引用断裂。

## 演化模型速记

```
[想法 / 需求]   起点：00_想法&诉求池/
      │
      ├─→ 演化为「项目」  (有始有终, 命名 10_项目/XMxx_xxx)
      ├─→ 演化为「产品」  (长期持续 + 产品资产, 命名 20_产品/CPxx_xxx)
      └─→ 消亡           (评估失败)
```

| 演化路径 | 操作 |
|---|---|
| 想法 → 归属某产品 | mv 到 `20_产品/CPxx_产品/22_需求卡片/`，原想法留指针标 `状态: 已转化` |
| 想法 → 自成独立项目 | mv 到 `10_项目/XMxx_xxx/`（frontmatter 标 `产品`，归属或独立）|
| 想法 → 升级为产品 | `20_产品/` 下新建 `CPxx_xxx/`，扩充产品资产，想法留指针 |
| 想法 → 消亡 | mv 到 `00_想法&诉求池/99_归档/已消亡/` |
