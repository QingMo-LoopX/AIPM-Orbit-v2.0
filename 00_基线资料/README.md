# 00_基线资料 · 历史资产 / 现行规则

> 进入 AIPM-Orbit 之前就已经存在的文档存放区。承接历史资产、现行业务规则、老架构、过往合同等"现状基线"。
> 与 [[00_想法&诉求池]] 是时间方向的对偶：**过去/现状基线 vs 未来/新想法**。

## 这里放什么

- 历史 PRD（已上线产品的旧版本规格）
- 现行业务规则（当前在执行的规则文档）
- 老架构图 / 老技术文档
- 过往合同 / SLA / 协议
- 重大决策类的既往会议纪要归档
- 任何"已存在、现在还在用、需要被引用"的基线材料

## 不放什么

| 内容 | 应该去 |
|---|---|
| 新提出的想法 / 诉求 | [[00_想法&诉求池]] |
| 在跑项目的过程产物 | `10_项目/XMxx_xxx/` |
| 当前迭代里产出的工件 | 对应产品的 `21_原始需求/` 等 |
| 通用方法论 / 知识 | `81_知识库/` |
| 研究类沉淀 | `80_研究沉淀/` |

> 边界粗判：是不是 AIPM-Orbit 启用之前就有？是 → 基线；否 → 不在这里。

## 内部组织（按产品/项目编号集中）

```
00_基线资料/
├── README.md             ← 本文件
├── 跨产品/                公司级 / 横跨多产品的 legacy
├── CP01_xxx/             具体产品的历史资产
├── CP02_xxx/
└── ...
```

子目录内部自由组织，推荐再按类型分：`历史PRD/`、`现行业务规则/`、`老架构/`、`合同与协议/` 等；或按时间分：`2023/`、`2024/`、`2025/`。

## 命名建议

文件名建议带版本/日期，方便溯源：

- `xxx-计费规则-V1.2-2024.md`
- `xxx-原架构图-2023Q4.png`
- `集团IT安全规范-2025年版.pdf`

## 行为约定

- **持续接收**：不仅是迁移一次，新发现的 legacy 文档随时入库
- **被引用就别随便改名**：其他文档可能 wiki-link 过来
- **被新版替代时归档不删除**：老版进 `历史/` 子目录，溯源不断
