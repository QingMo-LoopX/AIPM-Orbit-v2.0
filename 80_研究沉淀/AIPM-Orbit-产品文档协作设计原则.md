# AIPM-Orbit-产品文档协作设计原则

## 核心判断

- 产品经理最稳定、最高频、最容易标准化的工作流是：`原始需求 -> 需求卡片 -> 原型管理 -> PRD管理 -> 测试用例`
- 在这个阶段，最重要的不是大平台，而是把三类文档的关系跑顺

## 方法

- 用 `00_收件箱` 承接原始输入
- 用 `10_项目` 承接正式项目与三份核心文档
- 用 `40_知识库` 维护通用概念和规则
- 用 `99_系统/模板` 保证每次从统一结构开始

## 设计原则

- 先结构化，后成文
- 先追溯，后扩展
- 先单项目闭环，后多项目复用
- 先人工拍板，后自动化提效
