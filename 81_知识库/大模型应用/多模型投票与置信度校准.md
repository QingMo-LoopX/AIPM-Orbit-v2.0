# 多模型投票与置信度校准

## 核心思想

> **不要信大模型自己说的「我 0.95 置信」**——LLM 自报置信度普遍不准。

更可靠的置信度信号是 **「多次 / 多模型一致性」**：一致 = 高置信，分歧 = 低置信。

## 多模型投票（Ensemble）

并行调多个不同模型，少数服从多数。

```
工单/图 ┬→ qwen3-vl-plus     → 是
       ├→ qwen3-vl-flash    → 是    3:0 一致 → 高置信「是」
       └→ 其他备选 VLM      → 是
       
工单/图 ┬→ A → 是
       ├→ B → 否            1:1:1 分歧 → 转复核
       └→ C → 不好判断
```

**适用场景**：主观维度（如「是否清洁达标」），抑制单模型偏见 / 幻觉。

## 同模型多次采样（Self-consistency）

同一个模型对同一道题跑 N 次，每次温度（temperature）微调 / prompt 微变，取多数票。

```
qwen3-vl-plus × 5 次 → [是, 是, 是, 是, 否]   → 4:1 → 输出「是」
qwen3-vl-plus × 5 次 → [是, 否, 是, 否, 不好]  → 2:2:1 → 转复核
```

**适用场景**：边界 case 判定不稳——同一模型多次跑结果不一致，本身就是「模型也吃不准」的信号。

## 离散度 → 置信度信号

```
一致度高 → 高置信 → 走 AI 自动通过 / 不通过
一致度低 → 低置信 → 转复核
```

**关键阈值**（**待人工确认**，需技术评估后落定）：
- 跨模型投票几个模型？
- 同模型采样几次？
- 多大一致性算"高置信"？（如 N=5 时 4 票一致就算？还是要 5 票全一致？）
- 不同维度阈值是否一致？

阈值未定时**工程没法实现**，方案里要标 "**待技术评估**"。

## 置信度校准（Calibration）

模型原始打分（包括投票一致性）不能直接当置信度用，要**映射成与真实准确率一致**的校准置信度。

### 三阶段

**1. 初始校准（上线前）**
- 影子运行：用历史工单已有的人工质检结果当 ground truth
- 对账 AI 影子判定 → 标定各维度准确率
- 据此设定 自动通过阈值

**2. 持续校准（上线后）**
- 复核员实际判定回流当 ground truth
- 滚动校准准确率与阈值
- 不再依赖一次性的影子标定结果

**3. 数据不足时的兜底**
- 历史人工质检数据少 → **初始阈值从严**（自动通过门槛高、多转复核）
- 随数据积累再逐步放宽
- 避免冷启动阶段假阳性大量打扰一线

## 维度独立判定，互不连坐

各维度（如 ②③④⑤）**独立校准 + 独立判定**：一个维度判否不影响其他维度。

聚合时再按四态公式合成工单级结果。

这一点容易被误解——容易写出"② 判否 → ③④⑤ 跳过"的方案，实际是错的。每个维度都要判到底。

## 与「不好判断」三态的协同

校准置信度 + 三态判定 配套：
- 置信度 ≥ 高阈值 → 落「是 / 否」
- 置信度 < 高阈值 → 落「**不好判断**」 → 转复核

**不好判断** 是降级的合法出口，避免硬判提升表面准确率但代价是把 bad case 藏进"是"里。
