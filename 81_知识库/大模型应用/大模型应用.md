# 大模型应用

多模态 / 大语言模型在产品工程里的方法论沉淀。**与具体业务无关**，跨项目可复用。

## 入口

- [[VLM应用架构选型]] — 调 API vs 自建 RAG，标杆库设计
- [[VLM判定与受控词表]] — 端到端判定方式 + 三态判定 + 受控词表机制
- [[多模型投票与置信度校准]] — Ensemble + Self-consistency + 校准流水线
- [[VLM评测集设计]] — JSONL 评测集结构 + VlmEvalKit 集成 + 指标分层

## 适用场景

任何**把 LLM/VLM 作为业务能力**接入的产品方案。常见模式：

- 多模态判定（图片 + 文本 → 业务标签）
- 文本理解与生成（工单分类 / 摘要 / 改写）
- 复核辅助（AI 给初判 + 证据，人做终审）

## 反模式

- 把 LLM 当确定性 API：要让模型用受控词表输出，靠 fallback 兜底
- 一上来就自建 RAG：先用业务数据库做标杆库，跑通再升级
- 信模型自报置信度：靠多次/多模型一致性算
- 多维度合并成一道大题：数据层合并 ✓，任务层合并 ❌
