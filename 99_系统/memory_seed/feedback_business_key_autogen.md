业务 key 字段（角色编号 / 组织编号 / 项目编号 / 任何 `*_code`、`*_no` 类稳定标识）一律由系统自动生成。表单里这类字段：

- **新增态**：disabled，placeholder 显示「保存后系统自动生成」（不要写"自动生成 / 可编辑"这种含糊表述）
- **编辑态**：disabled，显示当前值
- 两态都不让人工填、不让人工改

**Why:** 业务 key 是系统内的稳定标识，被外键、wiki-link、缓存、报表、操作日志等大量引用。人工填会引入命名冲突、格式不一致、改名导致引用断裂等问题。卓越场景下用户多次反馈这条，属于稳定偏好。

**How to apply:**

- 做任何 CRUD 表单时，先识别哪些字段是业务 key（编号 / code / no / id 类），统一按此规则处理
- 列表页『新增』『编辑』入口对应的弹窗 / 详情页都要照办
- 不需要每次问，直接按此规则做
- 如果业务 key 不显示在表单（如项目配置弹窗里直接不带项目编号字段）也可以，只要保证人工拿不到改它的入口
