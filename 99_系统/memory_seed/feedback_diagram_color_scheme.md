画业务关系图 / 拓扑图 / 架构图时，**同一类对象用同一个颜色**，颜色变化只用来区分『不同概念类』。

## 反例（V3.2 之前）

主数据分层 4 个节点 / 4 个底色：
- 租户（卓越集团）橙色
- 公司（卓品）红色
- 项目（卓悦中心等）黄色
- 空间 蓝色
- 部门 浅蓝

这是『按层级编号』配色，制造视觉噪声 —— 它们本质是同一类（业务主数据 / 组织结构节点），用户看图时要先识别这些颜色又意识到颜色没语义，是浪费认知。

## 正确做法

主数据层（租户 / 公司 / 项目 / 空间 / 部门 / 任何组织结构节点）：**统一石板灰**（slate）色系
- 节点 fill: `#64748B` / `#94A3B8`
- 节点 stroke: `#475569` / `#334155`
- 背景 fill: `#F1F5F9`
- 背景 stroke: `#94A3B8`
- 标题文字: `#475569`
- 主轴 / 关系线 stroke: `#475569`

需要保留区分的概念类（真正不同的东西）：
- 企业用户 → 绿色 `#10B981`
- 客户用户 → 紫色 `#7C3AED`
- 平台角色 → 深红 `#DC2626`
- 项目角色 → 橙色 `#EA580C`
- N:N 跨层关系：绿（用户）/ 紫（客户）虚线
- 普通箭头：蓝色 `#1677FF`（保留作为通用方向指示，但主数据内部用 slate）

**Why:** 2026-05-14 在权限关系图 V3.3 里给『租户 / 公司 / 项目 / 空间』各上一种颜色，用户指出"这是一套业务主数据的不同分层，其实不需要分颜色，都是一个色"。每个层级独立染色 = 制造视觉噪声但不传递语义。

**How to apply:**
- 每次开始画图前先列出对象类别，识别哪些是『同一类 / 不同分层』，哪些是『不同概念类』
- 同类对象只允许一种主色（深浅可微调表强调，但色相一致）
- 不同概念类才换色
- 关系线：同类内部关系线也用同一色（slate），跨类关系线才用差异色
