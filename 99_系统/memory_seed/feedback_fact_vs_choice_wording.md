描述卓越业务里**客观存在**的事实时，用『形态』/『事实』/『现状』，**不要**用『选择』/『决定』。

典型场景：
- ✗ 卓越的选择：A 型（两条线 + 人工同步）
- ✓ 卓越的形态：A 型（两条线 + 人工同步）
- ✗ 卓越选择只维护项目组织
- ✓ 卓越只维护项目组织（事实就是这样）
- ✓ D1 决策：多项目场景上级关系跟项目走（这是真的决策，从多方案里挑的，可以用"决策""选择"）

**Why:** 『选择』隐含『我们从多个候选里挑了这个，理论上可以挑别的』。但卓越的两条线（HR 组织 + 工单项目组织 + 人为同步）是**客观现状** —— HR 系统本就独立存在，工单系统就是独立的项目组织。这不是产研拍板出来的，是业务事实。用『选择』会让读者以为这是可逆决策，引发『那能不能改成 B』之类的伪问题。2026-05-10 用户明确纠正：『卓越的选择』→『卓越的形态』。

**How to apply:**
- 『选择』/『决定』/『决策』留给真正的决策项（D1 / D2 那种从多个候选方案里挑了一个）
- 『形态』/『事实』/『现状』/『结构』用于描述客观存在的业务现实
- 写方案时自检：每个『选择』『决定』背后是不是真的有候选项？没候选就改词
