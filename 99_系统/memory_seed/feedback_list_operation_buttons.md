列表（用户管理、服务人员、项目配置等）行操作列里的按钮**始终全部展示**。当某行对某个按钮不适用时，**禁用**该按钮（灰化、不可点击），不要按行隐藏按钮。

**Why**：2026-05-18 用户管理设计讨论时明确——列宽对齐 + 能力可见性比『按需隐藏』更重要。业务用户看到的列表，每行结构应一致；如果某行某按钮缺失，反而会让人疑惑『为什么这一行没有？是 bug 吗？』。禁用状态既保留了能力存在的视觉信号，又表达了『此场景下不适用』。

**How to apply**：

- 设计 PRD / 原型时，列表的操作列按钮按『全集』规划，不要按用户类型 / 状态做行级隐藏
- 不同行的按钮可用性差异通过『禁用态』表达，禁用时建议加 hover 提示说明为什么不可用
- 适用场景：用户管理（企业 / 客户用户行的『组织授权』）、服务人员（在职 / 离职行的某些操作）、订单列表等任何"同列表多类型行"的场景
- 不适用场景：按钮在所有行都用不上时（直接整列不出现），或按钮数量极多需要折叠到『更多』菜单时（按场景动态折叠）
