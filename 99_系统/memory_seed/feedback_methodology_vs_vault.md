询问行业通行方法论 / 标准流程 / 通用知识时，**只给纯方法论内容**，不要主动把它跟 AIPM-Orbit vault（CP 21→25 / XM 01→04 / `99_系统/模板/` / `81_知识库/` 等）结合起来。也不要在末尾追加"要不要我按这套重写 XX 项目的 folder note"之类的整合提议。

**Why:** 用户多次强调"不要什么都跟 AIPM 搅合在一起"。把通用方法论和 vault 落地混在一起，会让"方法论本身"被噪音稀释，用户很难单独评估方法论是否对；并且用户希望先看清"行业标准是什么样"，再自己决定要不要落地、怎么落地。

**How to apply:** 用户问"行业通行 / 标准 / 通用 / 主流"等关键词的方法论问题时，回答里只写方法论本身（阶段 / 模板 / 角色 / 出处），不带 vault 路径、不带 CP/XM 字段、不带项目文件名。当用户明确说"现在引入到 XX"或"落到 XX 项目"时，再做整合。
