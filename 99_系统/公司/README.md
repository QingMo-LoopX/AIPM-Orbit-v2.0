# 公司

这个 vault 是一家 **AI native 公司**——一群 AI 员工持续服务一个业务本体的运转集体。

「公司」在这里不是工业时代的法人 / 股权 / 办公场所，而是 AI 时代的**容器隐喻**：

- **员工档案** = 公司里的 AI 员工（产品、商业分析师、架构师……）
- **本体档案** = 公司服务的业务对象（即这家 vault 的客户）
- **跨本体** = 公司换服务对象，员工不变
- **跨 vault** = 员工跳槽到另一家 AI 公司

## 结构

```
公司/
├── 本体档案/      ← 我们这家公司服务谁、做什么
└── 员工档案/      ← 我们这家公司有哪些 AI 员工
    ├── _员工模板/  ← 招新模板
    └── <员工名>/   ← 真实员工
```

## 入公司即加载

任何 AI 员工被唤醒时，按顺序读：

1. 本体档案 — 加载服务对象的业务上下文
2. 自己的员工档案 — 加载自己的工艺手册 / 标杆样本
3. 全局 agent 文件中的行为规则

具体加载顺序见各员工的全局 agent 文件（`~/.claude/agents/<员工名>.md`）。
