# 员工档案

本 vault 在职 AI 员工花名册：

| 员工 | 岗位 | 入职 |
|---|---|---|
| （首位员工） | （岗位） | YYYY |

每位员工自带：
- **README** — 个人介绍与档案索引
- **工艺手册** — 在本本体积累的方法论（条目可跨本体继承）
- **标杆样本** — 本本体认可的产物参考（不可跨本体迁移）

## 招新流程

1. **建岗位全局 agent 文件**：在 `~/.claude/agents/` 建该员工的全局 agent 定义文件，承载岗位通用能力（不写本本体特定内容）
2. **复制员工模板**：把 `_员工模板/` 复制为 `<员工名>/`
3. **填占位**：替换 `<员工名>/README.md` 里的 `{{员工名}}` `{{岗位}}` 占位
4. **入花名册**：在本 README 的花名册表格加一行
5. **首次入本体**：让员工先读 `99_系统/本体档案/` 和自己的 `工艺手册/`，确认能正确加载上下文

## 跨本体迁移

员工换 vault 时：
- **全局 agent 文件**：不动（岗位能力跨本体通用）
- **工艺手册**：可整套继承（方法论不带本体味）
- **标杆样本**：按新本体重新积累（每家口味不同，不可直接迁移）

## 沉淀机制（员工通用）

用户口令触发：
- 「**沉淀工艺**」→ 员工把刚才被纠正的内容归纳到对应工艺手册条目
- 「**记入标杆**」→ 员工把刚才认可的产物归档到对应标杆样本目录
