# 技能目录索引

> 配合 [[CLAUDE]] / [[AGENTS]] 中定义的 slash workflows 使用。

## 高频动作

| 命令 | 用途 |
|---|---|
| `/收集原始需求` | 把外部输入分流到对应位置（基线 / 想法池 / 产品 / 项目） |
| `/生成需求卡片` | 把原始需求提炼为结构化卡片（套用 [[需求梳理模板]]） |
| `/补原型` | 从需求卡片生成原型管理（套用 [[原型管理模板]]） |
| `/写PRD` | 从需求卡片和原型生成 PRD（套用 [[PRD模板]]） |
| `/生成测试用例` | 从 PRD 生成测试用例（套用 [[测试文档模板]]） |
| `/想法转化` | 把 `00_想法&诉求池/` 中已立项的想法转移到对应产品/项目 |
| `/项目立项` | 新建 `10_项目/XMxx_xxx/` 文件夹结构 + folder note（套用 [[Project_Template]]） |
| `/产品立项` | 新建 `20_产品/CPxx_xxx/` 文件夹结构 + folder note |
| `/归档` | 项目 / 产品 / 想法消亡时归档 |

## 使用建议

- **材料散乱时，先做分流**——别一上来就写 PRD
- **想法没确认前，不急着立项**——先在 `00_想法&诉求池/` 评估
- **PRD 没定稿前，不急着写测试用例**——避免返工
- **测试用例永远从已确认规则出发**——不从想象出发
- **每个工件 frontmatter 必填 `产品`**——用 `[[wiki-link]]` 不用纯字符串
