# 排序规则

> 本文件由 [Custom File Explorer Sorting](https://github.com/SebastianMC/obsidian-custom-sort) 插件读取。
> 规则：vault 根目录及所有子目录中，**文件和文件夹混排升序**。
> 置顶逻辑通过 **Bookmarks 分组 `sortspec`** 实现（详见下文）。

## 配置插件

打开 Obsidian 设置 → Custom File Explorer Sorting → "Path or name of additional note(s) containing sorting specification" 填入：

```
99_系统/排序规则
```

## 规则解读

- `target-folder: /` — 根目录
- `< a-z` — 升序混排（文件 + 文件夹按名称混合排序）
- `target-folder: /*` — 通配，递归到所有子目录
- 第二个 `< a-z` — 子目录内部同样混排升序

## 置顶机制（Bookmarks 集成）

文件树里想强制某些文件 / 文件夹**置顶**，使用 Obsidian 的 Bookmarks（核心插件）：

1. **启用** Core Bookmarks 插件：`设置 → Core plugins → Bookmarks` 打开
2. **创建分组** `sortspec`：在 Bookmarks 面板右键 → New group → 命名 `sortspec`
3. **添加置顶项**：右键文件 → `Custom sort:` → `Bookmark it for sorting` 自动加入 sortspec 分组
4. **拖动调整顺序**：在 Bookmarks 面板里拖 sortspec 分组里的项目，顺序即文件树置顶顺序
5. **应用**：`Cmd+P` → 输入 `sort on` → 选第一个命令（"Enable and apply the custom sorting"）

## 修改 / 失效排查

- 修改 sortspec 后跑 `sort on` 命令立即生效
- 不生效检查：插件设置里"Use sortspec.md files"或类似选项是否开启
- Bookmarks 路线不生效：检查 Bookmarks 面板里**有没有 `sortspec` 分组**、**置顶项是不是在该分组内**（不是其他分组）、**插件设置 Bookmarks integration 默认开启**

## 备用：文件名前缀置顶

如果 Bookmarks 路线不可用，可以通过文件名前缀强制置顶。例如把 `00_看板.md` 重命名为 `00_!看板.md`：
- ASCII `!` (0x21) 排在所有 `_` `0-9` `a-z` `A-Z` 中文之前
- 强制该文件出现在文件树最前
