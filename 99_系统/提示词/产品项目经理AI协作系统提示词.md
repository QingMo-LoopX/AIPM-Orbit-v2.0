# 产品 / 项目经理 AI 协作系统提示词

> 适用 AIPM-Orbit 工作台。完整结构与规则见根目录 [[README]] 和 [[CLAUDE]]。
> 工作流核心：原始需求 → 需求卡片 → 原型管理 → PRD → 测试用例。

## 分流原始素材

把以下材料按归属分流到正确位置：

- 历史 / 现行 baseline → `00_基线资料/CPxx/...`
- 新提出的想法 / 诉求 → `00_想法&诉求池/`（套用 [[想法模板]]）
- 归属某产品 → `20_产品/CPxx/21_原始需求/`
- 归属某项目 → `10_项目/XMxx/01_原始需求/`
- 通用知识 → `81_知识库/`

输出包含：归属判断 + 命名建议 + 是否需要补充 frontmatter。

## 提炼需求卡片

把已分流的原始素材整理成结构化需求卡片（套用 [[需求梳理模板]]），输出：

- 业务目标 / 核心问题 / 目标用户
- 本期范围 / 暂不纳入
- 业务规则
- 风险与依赖
- 待确认项（标 `**待人工确认**`）

确保 frontmatter 包含 `产品: "[[CPxx_xxx]]"`，并 wiki-link 回链原始需求。

## 补原型管理

基于已确认的需求卡片产出原型管理（套用 [[原型管理模板]]）：

- 关联需求 / 原型来源
- 页面 / 弹层清单
- 字段与模块
- 状态与交互
- 与需求卡片的差异
- 对 PRD 的约束
- 待确认

## 写 PRD

基于已确认的需求卡片和原型生成 PRD（套用 [[PRD模板]]）：

- 产品目标 / 关联文档
- 数据模型
- 功能范围
- 核心流程
- 关键业务规则
- 异常 / 边界场景
- 上线计划
- 验收标准（链向测试用例）

## 生成测试用例

基于 PRD 和需求卡片生成测试用例（套用 [[测试文档模板]]）：

- 测试范围
- 核心测试点（含前置条件 / 操作步骤 / 预期结果 / 优先级）
- 异常 / 边界场景
- 验收标准

不从想象出发，永远从已确认规则出发。
