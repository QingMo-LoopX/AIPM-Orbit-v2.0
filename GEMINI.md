# Gemini Context — AIPM-Orbit

你在 Obsidian 工作区 `AIPM-Orbit` 协作。这是 AI 产品经理的全流程工作台。

> 完整结构与规则见 [README.md](./README.md)（系统框架）和 [[00_AIPM-Orbit整体说明]]（本 tenant 业务）。Claude Code / 通用 Agent 规则见 [CLAUDE.md](./CLAUDE.md) / [AGENTS.md](./AGENTS.md)。

## 核心目标

- 把原始素材分流到正确位置（基线 / 想法池 / 产品 / 项目）
- 在产品/项目内部按 pipeline 推进：
  - **产品级（CP）**：原始需求 → 需求卡片 → 原型管理 → PRD管理 → 测试用例（21~25）
  - **项目级（XM）**：原始需求 → 需求卡片 → 业务建模 → 方案设计（01~04，XM 内不出 PRD/原型/测试）
  - XM 方案验证完成后，业务建模 / 方案设计成果回流到所属 CP 的 1X 架构层
- 输出可追溯文档：frontmatter `产品: "[[CPxx_xxx]]"` + `[[wiki-link]]` 回链上游

## 顶层路径速记

- `00_基线资料/` — 历史 / 现行 baseline
- `00_想法&诉求池/` — 新想法演化起点
- `20_产品/CPxx_xxx/` — 长期产品
- `10_项目/XMxx_xxx/` — 有始有终的项目
- `30_迭代管理/` `40_线上问题/` — 横切运行期
- `80_研究沉淀/` `81_知识库/` — 横切参考

## 工作原则

- 不空谈愿景，优先产出可落地文档
- 不默认全自动，关键规则和验收项必须 `**待人工确认**`
- 不扩展到研发排期、缺陷管理等非边界内容
- 中文输出
- frontmatter 后不空行；关联类字段用 `[[wiki-link]]` 不用纯字符串
